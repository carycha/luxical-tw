# Product Guidelines
## 寫作風格 (Prose Style)
- **清晰與精準：** 技術文檔應簡潔，並使用精確的學術和技術術語。
- **語氣：** 探討方法論時採用專業、直接且學術的風格；提供教程時採用實用且對開發者友好的語氣。
- **語言：** 程式碼和正式文檔以英文為主，專案管理溝通則使用繁體中文。

## 品牌形象 (Branding)
- **名稱：** Luxical (首字母大寫)。
- **Logo：** 使用專案目錄下的 `Luxical_logo.png`。
- **審美：** 簡潔、極簡且帶有「快速」感 (例如使用「lux」、「light」等相關意象)。

## 使用者體驗原則 (UX Principles)
- **效率優先：** 嵌入過程應盡可能快速，並減少計算開銷。
- **簡約設計：** API 應該直觀；複雜的內部邏輯應對終端使用者進行抽象化處理。
- **可靠性：** 錯誤訊息應具備充足的資訊，有助於排查效能或配置問題。
- **透明度：** 明確說明支援的平台（Linux、MacOS）以及任何硬體/軟體的限制。

## 文檔結構 (Documentation Structure)
- 以 README 作為主要的專案入口點。
- 為所有公開 API 方法提供內嵌式 Docstrings。
- 提供完整的訓練與推理範例。
